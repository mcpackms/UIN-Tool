## 应用概述

UIN Tool 是一个轻量级的插件化框架，允许您动态加载和运行第三方插件。插件可以是：

| 插件类型 | 说明 | 适用场景 |
|----------|------|----------|
| **原生插件** | 使用 Java 代码开发，性能最优 | 需要高性能、复杂交互的功能 |
| **Web插件** | 使用 HTML/CSS/JS 开发，快速迭代 | 界面丰富、更新频繁的功能 |

### 主要功能

| 功能 | 说明 |
|------|------|
| 🚀 启动体验 | SplashActivity 启动页、首次启动引导页 |
| 🔗 应用快捷方式 | 长按应用图标快速进入功能页面 |
| 🔄 自动更新 | 启动自动检测新版本，支持忽略版本 |
| ⚡ GitHub加速 | 镜像站管理、CDN加速、自定义镜像站 |
| 📦 插件仓库 | 从 GitHub 官方仓库浏览、搜索和安装插件 |
| 🔌 插件管理 | 导入、导出、卸载插件，支持单文件、批量、插件集导入 |
| 🛠️ 开发工具 | 创建插件向导、导出开发模板 |
| 🔐 权限管理 | 统一管理应用权限和插件独立权限，含权限说明 |
| 💾 备份恢复 | 备份/恢复所有插件和配置 |
| 📋 运行日志 | 查看、导出、清空运行日志和崩溃报告 |
| 🖥️ 桌面小部件 | 在桌面添加快速启动小部件（列表小部件、1x1快捷方式） |
| 🎨 UI个性化 | 自定义主题颜色（38+颜色项）和圆角大小，图标着色开关 |
| 📚 文档中心 | 开发文档、使用帮助、更新日志、关于信息、贡献者 |
| 🌐 网络请求 | Web插件支持 HTTP GET/POST 请求 |
| 📁 文件系统 | Web插件支持文件读写操作 |
| 📡 传感器 | Web插件支持加速度计、陀螺仪等传感器 |

## 首次启动

### 启动流程

```

用户点击图标 → SplashActivity (启动页) → 检查首次启动/版本更新
├── 是 → OnboardingActivity (引导页) → MainActivity
└── 否 → MainActivity

```

### 启动页 (SplashActivity)

应用启动时会显示启动页，包含：
- 应用 Logo
- 应用名称
- 版本号

启动页会进行必要的初始化工作，然后自动跳转到主页面。

### 引导页 (OnboardingActivity)

首次安装或版本更新时，会自动显示引导页，介绍核心功能：

| 页面 | 内容 |
|------|------|
| 第1页 | 欢迎语 / 版本更新说明 |
| 第2页 | 插件管理功能介绍 |
| 第3页 | 插件开发工具介绍 |
| 第4页 | Web 插件支持介绍 |
| 第5页 | 准备就绪，开始体验 |

**操作方式：**
- 点击「下一步」翻页
- 点击「跳过」直接进入主界面
- 最后一页点击「开始体验」

## 自动更新

> 🆕 **v3.4.0 新增功能**

### 启动自动检测

每次启动应用时，系统会自动检查 GitHub 最新版本。

### 强制更新机制

> 🆕 **v3.10.0 新增功能**

当 Release Tag 格式为 `{版本代码}-{版本名称}-1` 时，会强制用户更新：

| Tag 格式 | 示例 | 强制更新 |
|----------|------|----------|
| `x-y-z` 且 `z=1` | `2-1.0.1-1` | ✅ 是 |
| `x-y-z` 且 `z=0` | `2-1.0.1-0` | ❌ 否 |
| `x-y` | `2-1.0.1` | ❌ 否 |

**强制更新特点：**
- 弹窗中「暂不更新」按钮被隐藏
- 用户必须选择「自动下载」或「手动下载」
- 无法跳过更新

### 发现新版本

当检测到新版本时，会弹出更新对话框：

```

┌─────────────────────────────────────┐
│  📱 发现新版本                       │
├─────────────────────────────────────┤
│  版本号：3.10.0                      │
│  版本代码：9                         │
│  发布日期：2026-06-09                │
│  文件大小：5.2 MB                    │
│                                     │
│  更新内容：                          │
│  • 新增 GitHub 加速功能              │
│  • 新增强制更新机制                  │
│  • 修复多个 Bug                      │
│                                     │
│  请选择更新方式：                    │
├─────────────────────────────────────┤
│ [自动下载] [手动下载] [暂不更新]     │
│ (强制更新时无"暂不更新"按钮)          │
└─────────────────────────────────────┘

```

### 更新选项

| 选项 | 说明 |
|------|------|
| **自动下载** | 在应用内下载 APK，完成后直接安装 |
| **手动下载** | 打开浏览器跳转到 GitHub Releases 页面 |
| **暂不更新** | 关闭对话框，正常进入应用（普通更新可用） |
| **忽略此版本** | 记录忽略的版本，下次启动不再提示（普通更新可用） |

### 版本忽略

如果选择「忽略此版本」，该版本将不再提示更新，直到有更新的版本发布。

### 检查更新失败

当网络问题导致检查更新失败时，会显示手动下载选项，用户可自行前往 GitHub 下载。

## GitHub加速

> 🆕 **v3.10.0 新增功能**

GitHub加速功能帮助国内用户更快地访问 GitHub 和下载插件。

### 访问方式

1. 点击底部「管理」标签
2. 点击「GitHub加速」卡片
3. 进入镜像站管理页面

### 镜像站管理

#### 内置镜像站

系统内置了 13+ 个国内可用的镜像站：

| 镜像站名称 | URL |
|-----------|-----|
| GitHub 官方 | https://github.com |
| FastGit | https://hub.fastgit.xyz |
| GhProxy | https://ghproxy.net |
| Mirror GhProxy | https://mirror.ghproxy.com |
| Moeyy | https://github.moeyy.xyz |
| GitClone | https://gitclone.com |
| GhApi | https://gh.api.99988866.xyz |
| Jiewen | https://gh.jiewen.ltd |
| CDN Proxy | https://ghproxy.ltd |
| 加速 GitHub | https://github.zhlh6.cn |
| Kkgithub | https://kkgithub.com |
| Huu | https://hub.uuu.moe |
| FastGit 下载 | https://download.fastgit.xyz |

#### 启用/禁用镜像站

1. 在镜像站列表中勾选要启用的镜像站
2. 勾选的镜像站会在更新检查时自动测试和使用
3. 系统会选择最快的可用镜像

#### 全选/取消全选

- 点击「全选」按钮：勾选所有镜像站
- 点击「取消全选」按钮：取消所有勾选

#### 添加自定义镜像站

1. 点击「添加」按钮
2. 填写镜像站名称和 URL
3. 可选填写备注
4. 点击「添加」保存

#### 导入镜像站列表

1. 点击「导入」按钮
2. 选择 TXT 格式的镜像站列表文件
3. 文件格式：每行 `名称|URL|备注`

#### 导出镜像站列表

1. 点击「导出」按钮
2. 系统生成当前镜像站列表
3. 可通过分享功能保存

#### 重置镜像站

1. 点击「重置」按钮
2. 确认重置操作
3. 恢复为系统内置的默认镜像站列表

#### 测试可用性

1. 点击「测试可用性」按钮
2. 系统逐一测试每个镜像站
3. 测试结果显示在列表中（绿色✓表示可达，红色✗表示不可达）

#### 删除自定义镜像站

- 长按要删除的自定义镜像站
- 确认删除操作

### CDN加速

| 状态 | 效果 |
|------|------|
| 开启 | 使用 CDN 代理加速下载 |
| 关闭 | 直接使用原始 GitHub 地址下载 |

**操作方式：**
- 点击 CDN 加速卡片切换开关
- 卡片显示「已开启」或「已关闭」状态

### 保存设置

配置完成后，点击「保存设置」按钮：
- 保存镜像站启用状态
- 保存 CDN 加速开关状态
- 通知更新检查器使用新的配置

## 插件仓库

插件仓库允许您直接从 GitHub 官方仓库浏览和安装插件。

### 浏览插件

1. 点击底部「仓库」标签
2. 系统自动加载 GitHub 上的所有可用插件
3. 每个插件卡片显示：
   - 插件名称
   - 插件 ID
   - 版本号
   - 文件大小
   - 更新日期
4. 点击卡片可查看详细信息

### 搜索插件

1. 点击仓库页面的搜索框
2. 输入关键词（支持插件名称、ID、描述、作者）
3. 实时显示搜索结果
4. 点击清除按钮清空搜索

### 安装插件

1. 在仓库列表中找到要安装的插件
2. 点击「安装」按钮
3. 系统自动下载并安装
4. 下载进度实时显示：
   - 下载百分比
   - 已下载大小 / 总大小
5. 安装完成后按钮变为「打开」

### 打开已安装插件

1. 已安装的插件显示「打开」按钮
2. 点击「打开」直接运行插件

### 插件详情

点击插件卡片可查看详细信息：
- 完整插件名称
- 插件 ID
- 版本信息
- 文件大小
- 更新日期
- 作者信息
- 详细描述
- 更新日志

### 镜像加速

为了提升国内用户的下载速度，插件仓库内置了多个 GitHub 镜像站，系统会自动测试并选择最快的可用镜像。

> 💡 **提示**：可在「管理」→「GitHub加速」中自定义镜像站配置。

### 下拉刷新

如果插件列表没有更新，可以：
1. 在仓库页面向下滑动
2. 松手后自动刷新
3. 系统重新从 GitHub 获取最新插件列表

## 插件管理

### 插件格式说明

插件文件扩展名为 `.tpk`，本质上是 ZIP 压缩包。

#### 原生插件结构

```

plugin.tpk
├── plugin.json      # 插件配置文件
├── icon.png         # 插件图标 (建议 128x128)
├── plugin.dex       # 编译后的 DEX 文件
├── src/             # Java 源码目录
└── res/             # 资源文件目录

```

#### Web 插件结构

```

plugin.tpk
├── plugin.json      # 插件配置文件
├── icon.png         # 插件图标 (建议 128x128)
└── web/             # Web 资源目录
├── index.html   # 入口页面 (必须)
├── style.css    # 样式文件 (可选)
└── script.js    # JavaScript 文件 (可选)

```

### plugin.json 配置说明

```json
{
    "pluginId": "com.example.myplugin",      // 插件唯一ID (必填)
    "version": 1,                             // 版本号 (数字)
    "versionName": "1.0.0",                  // 版本名
    "minHostVersion": 1,                      // 最低宿主版本
    "name": "我的插件",                       // 插件名称 (必填)
    "author": "开发者",                       // 作者
    "description": "插件描述",                // 描述
    "icon": "icon.png",                       // 图标文件名
    "mainClass": "com.example.MainPlugin",    // 主类 (原生插件必填)
    "apiLevel": 21,                           // API级别
    "uiType": "native",                       // UI类型: native/web
    "entry": "web/index.html"                 // 入口文件 (Web插件)
}
```

导入插件

单文件导入

1. 点击底部「管理」
2. 点击「插件管理」
3. 点击「导入插件」按钮
4. 选择 .tpk 文件
5. 等待导入完成并查看结果

批量导入

1. 点击「批量导入」按钮
2. 选择多个 .tpk 文件
3. 系统会依次导入并显示进度
4. 导入完成后显示成功/失败统计

插件集导入

1. 点击「插件集」按钮
2. 选择包含多个 .tpk 文件的 ZIP 包
3. 系统自动解压并导入所有插件
4. 适用于批量备份恢复

导出插件

1. 在插件列表中勾选要导出的插件（支持多选）
2. 点击「全选」可快速选择全部
3. 点击「导出」按钮
4. 插件被打包为 ZIP 文件保存在工作目录
5. 文件名格式：Plugins_时间戳.zip

卸载插件

单个卸载

1. 点击插件进入详情页
2. 点击「卸载」按钮
3. 确认卸载

批量卸载

1. 勾选要卸载的插件
2. 点击「卸载」按钮
3. 确认批量卸载
4. 显示卸载结果统计

插件分类管理

修改分类

1. 点击插件进入详情页
2. 点击「修改分类」
3. 选择已有分类
4. 或选择「新建分类」创建新分类
5. 确认保存

删除分类

1. 在修改分类时选择「新建分类」
2. 在分类管理对话框中长按要删除的分类
3. 确认删除
4. 该分类下的插件自动移动到"未分类"

插件开发

创建原生插件

1. 点击「开发」→「创建新插件」
2. 选择「原生代码 UI」
3. 填写插件信息：
   · 插件ID：唯一标识，如 com.example.myplugin
   · 插件名称：显示名称
   · 作者：开发者名称
   · 描述：功能描述
   · 版本号：数字版本
   · 版本名：显示版本
   · 主类名：完整包名+类名
4. 选择插件图标（可选）
5. 编写 Java 代码
6. 添加资源文件（可选）
7. 生成项目文件

创建 Web 插件

常见问题

1. 如何获取更多插件？

· 📦 从「仓库」页面直接安装官方插件
· 🔧 自己开发插件
· 👥 社区分享

2. 插件安全吗？

· ✅ 插件导入时进行 SHA-256 签名验证
· ✅ 插件权限独立控制
· ⚠️ 建议只从可信来源获取插件

3. 工作目录在哪里？

默认路径：/storage/emulated/0/UIN_Tool/

```
UIN_Tool/
├── backups/       # 备份文件
├── downloads/     # 下载的 APK 文件
├── logs/          # 日志文件
├── [插件ID]/      # 插件目录
└── *.tpk          # 导入的插件文件
```

4. 如何修改工作目录？

当前版本不支持修改，使用默认路径。

5. 插件占用多少空间？

类型 大小
简单插件 50-200 KB
复杂插件 1-5 MB
Web插件 主要取决于资源文件

6. 支持在线更新吗？

· 插件仓库支持浏览和下载最新版本
· 应用本身支持自动更新检测
· 插件自动更新功能开发中

7. 如何反馈问题？

1. 导出运行日志
2. 截图错误信息
3. 发送到开发者邮箱或 GitHub Issues

8. 应用闪退怎么办？

1. 重新打开应用（会自动显示崩溃日志）
2. 查看运行日志分析原因
3. 清除应用数据
4. 重装应用

9. Web 插件不显示？

检查项：

· web/index.html 是否存在
· HTML 是否有语法错误
· JavaScript 是否有报错
· 查看运行日志

10. 如何更新 UIN Tool 本身？

· 方式一：应用启动自动检测新版本，选择自动下载
· 方式二：在「管理」→「检查更新」中下载安装
· 方式三：从 GitHub Releases 下载 APK 安装

11. 插件导入失败？

可能原因：

· 签名验证失败
· plugin.json 格式错误
· 缺少必要文件
· 存储空间不足

12. 小部件不显示？

· 检查小部件是否已添加
· 尝试重新添加
· 检查是否有已安装插件
· 刷新小部件列表

13. 如何查看版本更新内容？

点击「管理」→「文档中心」→「更新日志」

14. 如何管理插件分类？

在插件详情弹窗中点击「修改分类」：

· 选择已有分类
· 选择「新建分类」创建新分类
· 在分类管理对话框中可长按删除自定义分类

15. 仓库页面加载失败怎么办？

· 检查网络连接
· 下拉刷新重试
· 检查 GitHub 是否可访问
· 镜像站会自动切换
· 可在「管理」→「GitHub加速」中配置镜像站

16. 下载插件速度慢怎么办？

· 仓库页面会自动选择最快的镜像站
· 可在「管理」→「GitHub加速」中手动配置镜像站
· 开启 CDN 加速
· 可以尝试切换网络（WiFi/移动数据）
· 稍后重试

17. Web 插件网络请求失败？

· 检查 URL 是否正确
· 确认网络连接正常
· 查看日志中的错误信息

18. 传感器没有数据？

· 检查设备是否支持该传感器
· 确认已授予传感器权限
· 查看日志中的传感器状态

19. 文件读写失败？

· 确认文件名不包含非法字符
· 检查存储权限
· 确认磁盘空间充足

20. 检查更新失败？

· 检查网络连接
· 确认 GitHub 可访问
· 可在「管理」→「GitHub加速」中配置镜像站
· 稍后重试
· 查看日志中的详细错误

21. 首次启动有引导页吗？

是的，首次安装或版本更新时会自动显示引导页，介绍核心功能。

22. 应用启动有白屏怎么办？

v3.0.0 已优化白屏问题，请升级到最新版本。

23. 如何自定义图标颜色？

在「管理」→「UI个性化」中：

· 调整「图标着色」颜色值
· 使用图标着色开关控制是否应用主题色

24. 长按应用图标没有快捷方式？

· 需要 Android 7.1+ 系统
· 部分第三方启动器可能不支持
· 尝试使用系统默认启动器

25. 自动更新弹窗不显示？

· 检查网络连接
· 确认 GitHub 可访问
· 检查是否已忽略该版本
· 查看日志中的详细错误

26. 开发页面的按钮不显示怎么办？

v3.5.0 已修复此问题，请升级到最新版本。

27. 什么是强制更新？

当 Release Tag 格式为 {版本代码}-{版本名称}-1 时，会强制用户更新，无法跳过。这是为了确保用户使用最新版本，修复严重 Bug 或安全漏洞。

28. GitHub 加速功能怎么用？

在「管理」→「GitHub加速」中：

· 勾选要启用的镜像站
· 开启 CDN 加速
· 可添加自定义镜像站
· 可导入/导出镜像站列表
· 点击「保存设置」生效
